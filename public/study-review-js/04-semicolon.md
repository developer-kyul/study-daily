# JavaScript 세미콜론 논쟁

## 문장 끝엔 항상 세미콜론 추가해야 해*!*

일부 도서 또는 튜토리얼은 모든 JavaScript 줄 끝에 세미콜론(`;`)을 추가하는 것을 권장합니다.

```jsx
console.log('문장 끝엔 항상 세미콜론을 추가해야 합니다.');
```

어떤 사람들은 코드에 세미콜론을 추가하지 않으면 "형편없는 개발자"라고 말하기도 합니다.

항상 세미콜론을 명시적으로 사용하는 것이 안전하다고 생각하기 때문입니다.

**세미콜론 사용 찬성 측 주장 (장점)**

- 명시적이고 예측 가능함
- ASI의 예상치 못한 작동 방지
- 일부 코딩 스타일 가이드에서 권장
- 코드 압축 시 안전함

## 하지만 세미콜론은 선택사항인데

JavaScript는 세미콜론을 자동으로 삽입(ASI)합니다. 따라서 세미콜론을 추가하는 것은 선택 사항입니다.

> **ASI : Automatic Semicolon Insertion**
> JavaScript 엔진이 특정 조건에서 자동으로 세미콜론을 삽입하는 기능

### ASI가 작동하는 조건

JavaScript 엔진은 다음 상황에서 자동으로 세미콜론을 삽입합니다:

- 문장 끝에 세미콜론(`;`)이 없어도 문장이 중료되었다고 판단하는 경우
- `return`, `throw`, `break,` `continue` 등 키워드 다음에 줄바꿈된 경우 ( ⚠️ 문제 발생 가능)
- 블록(`}`)이 끝났을 때

### **ASI의 문제점과 해결 방법**

ASI는 편리하지만, 예상치 못한 작동을 일으킬 수 있습니다.

```jsx
let message = '안녕!'`JavaScript`.split('').forEach((c) => console.log(c));
// ❌ Uncaught TypeError: "안녕!" is not a function
```

문제를 해결하려면 세미콜론이 필요합니다.

```jsx
let message = '안녕!'**;** // ✅ 세미콜론 추가

`JavaScript`.split('').forEach((c) => console.log(c))
```

### 세미콜론이 꼭 필요할까?

좋은 JavaScript를 작성하는 데 세미콜론이 꼭 필요한 것은 아닙니다.

사실 세미콜론을 사용하지 않으면 JavaScript를 읽고 쓰기가 더 쉽습니다.

많은 JavaScript 전문가들은 이미 그렇게 하고 있습니다.

- Vue.js - Evan You
- npm - Isaac Z. Schlueter
- Standard JS - Feross Aboukhadijeh
- 많은 오픈소스 프로젝트들

**세미콜론 사용 반대 측의 주장**

사실 "항상 세미콜론을 쓰자."는 그렇게 간단한 이야기가 아닙니다.

실제로 세미콜론을 사용하지 말아야 하는 예외 상황들이 꽤 있어요.

```jsx
function foo() {
  return 42; // ✅ 세미콜론 필요
}**;** // ❌ 세미콜론 불필요 (함수 선언문 뒤)
```

```jsx
var foo = function () {
  // ...
}; // ✅ 세미콜론 필요 (변수 선언이므로)
```

그리고 아래와 같은 경우도 있죠.

```jsx
class Foo {
  constructor () {
    if (baz) {
      return 42; // ✅ 세미콜론 필요
    }**;** ❌ 세미콜론 불필요 (블록 끝)
    return 12;   // ✅ 세미콜론 필요
  }**;** // ❌ 세미콜론 불필요 (메서드 선언)
}**;** // ❌ 세미콜론 불필요 (클래스 선언)
```

### 세미콜론 없이 코딩하기

결론적으로 **"항상 세미콜론을 쓰자"**를 지키려면 **신경 써야 할 예외 상황이 엄청 많습니다**.

반대로 **"세미콜론을 절대 쓰지 말자"**는 주장에 따르면 **기억해야할 건 딱 하나**입니다.

> 핵심 규칙 : 절대 `[`, `(`, ``` 으로 줄을 시작하지 말자.

만약 `[`, `(`, ``` 으로 줄을 시작해야 한다면? 줄 앞에 세미콜론 하나 붙여주면 끝입니다.

```jsx
// [, (, `로 시작할 경우, 앞에 세미콜론 붙이면 문제 해결
[1, 2, 3].forEach(bar);
```

하지만 솔직히 이렇게까지 코드를 짜는 건 굳이 복잡하게 만드는 것일 수도 있어요.

변수에 할당하면 훨씬 더 깔끔하고 자연스럽게 쓸 수 있습니다. (권장)

```jsx
// 가장 자연스러운 방식
const nums = [1, 2, 3];
nums.forEach(bar);
```

💡

**다른 연산자는 어떨까?**

`+`, `*`, `/`, `-`, `,` 같은 연산자도 주의해야 한다고 말하지만,
실제 코드 상에서는 이걸 줄 맨 앞에 두는 경우가 거의 없습니다.

```jsx
// 이런 코드는 거의 작성하지 않음
let result = 1 + 2 + 3;

// 또는 여러 줄로 나눌 때
let result = 1 + 2 + 3;

// 일반적인 작성 방식
let result = 1 + 2 + 3;
```

# 세미콜론 논쟁 비교표

| 구분              | 세미콜론 사용             | 세미콜론 사용 안함     |
| ----------------- | ------------------------- | ---------------------- |
| **명시성**        | ✅ 명확함                 | ⚠️ ASI에 의존          |
| **예외 상황**     | ❌ 많음 (함수, 클래스 등) | ✅ 적음 (3가지만 주의) |
| **가독성**        | ⚠️ 시각적 노이즈          | ✅ 깔끔함              |
| **안전성**        | ✅ 예측 가능              | ⚠️ 규칙 숙지 필요      |
| **코딩 스타일**   | 전통적                    | 현대적                 |
| **유명 프로젝트** | jQuery, React             | Vue, npm, Standard     |

# 팀 프로젝트에서의 선택

### 중요한 것은 일관성

개인 프로젝트든 팀 프로젝트든, **일관성**이 가장 중요합니다.

**✅ 좋은 접근**

- 팀 전체가 하나의 스타일 가이드 합의
- 코드 리뷰에서 스타일 논쟁 최소화
- ESLint + Prettier로 자동화

**❌ 나쁜 접근**

- 파일마다 다른 스타일
- 개인 취향에 따라 변경
- 린터 없이 수동 관리

### 유명 스타일 가이드

**세미콜론 사용**

- [Google JavaScript Style Guide](https://google.github.io/styleguide/jsguide.html)
- [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript)

**세미콜론 미사용**

- [Standard JS](https://standardjs.com/)
- [Vue.js Style Guide](https://ko.vuejs.org/style-guide/)

# 요약

학습한 내용을 간단히 요약해봅니다.

**세미콜론 사용 찬성**

- ✅ 전통적인 접근 방식
- ✅ 명시적이고 예측 가능
- ❌ 예외 상황이 많음 (함수, 클래스 등)

**세미콜론 사용 반대**

- ✅ 깔끔하고 읽기 쉬움
- ✅ 단순한 규칙 하나만 기억 (`[`, `(`, ``` 주의)
- ❌ ASI 작동 이해 필요

**결론**

- 어느 쪽을 선택하든 일관성이 중요
- 린터(ESLint)를 사용하면 걱정 불필요
- 팀 프로젝트에서는 팀 규칙 따르기
- 개인 프로젝트에서는 편한 방식 선택
