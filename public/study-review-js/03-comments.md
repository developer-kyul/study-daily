# 주석

```jsx
// 한줄 주석

/*
여러줄
주석
*/

/**
 * 줄 맞추기 좋은
 * 여러 줄 주석
 */
```

# 주석 단축키

| 동작                  | Windows/Linux      | macOS                |
| --------------------- | ------------------ | -------------------- |
| **한 줄 주석 토글**   | `Ctrl + /`         | `Cmd + /`            |
| **여러 줄 주석 토글** | `Shift + Alt + A`  | `Shift + Option + A` |
| **블록 주석**         | 선택 후 `Ctrl + /` | 선택 후 `Cmd + /`    |

# 주석 요약

| 항목             | 내용                                                 |
| ---------------- | ---------------------------------------------------- |
| **한 줄 주석**   | `// 주석 내용`                                       |
| **여러 줄 주석** | `/* 주석 내용 */`                                    |
| **JSDoc 주석**   | `/** @param {type} name - 설명 */`                   |
| **주석 토글**    | `Ctrl + /` (Win) / `Cmd + /` (Mac)                   |
| **블록 주석**    | `Shift + Alt + A` (Win) / `Shift + Option + A` (Mac) |

| **나쁜 주석**    | 코드 반복, 오래된 주석, 불필요한 설명                |
| **원칙**         | 주석보다 좋은 코드 작성이 우선                       |

# 주석이란

문제를 해결하는데 도움이 되는 추가 메모를 작성할 때 주석을 사용

주석의 용도

- 코드 설명 및 문서화
- 디버깅 시 ,임시로 코드 비활성화
- 할일 (todo) 목록 작성
- 팀원과의 소통
- 학습 노트 작성

주석은 2가지

한줄 주석 **`//`**

여러줄 주석 **`/* */`**

## **여러줄 주석**

함수 문서화(JSDoc 스타일) - 주석을 넣어두면 코드를 작성할 때 팝업창으로 문서화되어 있는 해당 내용이 나옴

긴설명이 필요할때(여러줄 비활성화)

라이선스 정보

## **주석 작성 사례**

### **좋은 주석**

- 왜(why-왜 이 코드를 사용하는지)를 설명하는 주석
- 복잡한 로직 설명
- 경고(해당 코드의 값을 변경하면 안됨! 같은) 및 주의사항
- TODO와 FIXME

### **안좋은 주석**

- 명확한 코드를 반복 설명(i++ //변수 i를 1씩 증가시킴)
- 오래된 주석(코드와 일치하지 않는 주석, 이름 변경인데 주석은 나이 변경이라고 되어 있음)
- 불필요한 주석(함수시작, 변수 선언, 반환)
- 주석으로 코드 설명 대신 리팩토링(복잡한 모드를 주석으로 설명-성인이고, 프리미엄 회원이며, 활성 상태인지 확인)→해결법: 코드를 분리해서 작성하면 보기 편함

### 올바른 주석 사용법 :

주석을 사용하기보다 코드를 기능으로 나누고 명시적으로 작성해서 주석 없이도 이해되는 코드를 작성하는것이 좋습니다.
