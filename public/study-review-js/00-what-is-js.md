# 컴파일언어와 인터프리터 언어

## 프로그래밍 언어와 기계어

기계어 : 0과 1로 이루어진 언어

프로그래밍 언어 : 사람이 컴퓨터에게 지시를 내리기 위한 언어

## 고급언어와 저급언어

고급언어 : 사람들이 이해하기 쉬운 프로그래밍 언어 ( javascript )

저급언어: 컴퓨터가 이해하기 쉬운 언어 ( 어셈블리어 )

## 컴파일러 란?

사람이 이해하기 쉬운 고급언어를 저급언어로 바꿔주는 작업

즉 컴파일러는 사람과 컴퓨터 사이의 통역가

## 컴파일 언어와 인터프리터 언어

컴파일 언어 : 모든 코드를 컴파일 한 후 실행한다.

인터프리터 언어 : 한 줄 씩 작업이 끝날 때 마다 컴파일 작업후 실행 한다.

- 컴파일 언어 와 인터프리터 비교 표

| 항목              | **컴파일러 (Compiler)**                      | **인터프리터 (Interpreter)**              |
| ----------------- | -------------------------------------------- | ----------------------------------------- |
| 💡 작동 방식      | 전체 코드를 한꺼번에 번역해서 실행 파일 생성 | 코드를 한 줄씩 읽고 바로 실행             |
| 🕐 실행 속도      | 빠름 (번역은 느려도, 실행은 빠름)            | 느림 (매번 해석하면서 실행하니까)         |
| 🛠️ 에러 처리      | 코드를 다 번역한 뒤에 에러를 알려줌          | 한 줄 실행하다가 에러 나면 바로 멈춤      |
| 📁 출력 결과      | 실행 가능한 파일(.exe 등) 생성               | 별도의 실행 파일은 없음                   |
| 📘 예시 언어      | Java, C, C++, Go, Swift 등                   | JavaScript, Python, Ruby 등               |
| 🔁 반복 실행      | 빠름 (이미 번역된 파일을 실행하니까)         | 느림 (매번 새로 해석해야 하니까)          |
| 💬 개발 중 편리함 | 덜 편리함 (전체 코드 다시 컴파일해야 함)     | 편리함 (바로바로 실행해서 결과 확인 가능) |
| 🧩 비유           | 책 전체를 다 번역해서 새로운 책 만들기 📘    | 책을 한 줄씩 읽으면서 즉시 해석하기 📖    |

# **자바스크립트란?**

스크립트 언어-스스로 실행될 수 없다
실행되려면 웹브라우저와 같이 호스트 환경이 필요

🏷️ 크로스 플랫폼이란 ?

하나의 소스 코드나 동일한 방식으로 개발된 소프트웨어(앱, 게임 등)가 iOS, 안드로이드, Windows, macOS, 웹 등 다양한 운영체제나 하드웨어 환경(플랫폼)에서 모두 실행될 수 있는 특성 또는 개발 방식을 의미

🏷️ 크로스 플랫폼의 장점

자바스크립트의 장점은 모든 운영체제의 웹 브라우저에서 실행됨(HTML-구조설계,CSS-스타일도 마찬가지)

자바스크립트는 사용자와의 상호작용 목적으로 사용

## **자바스크립트로 할 수 있는 것**

- 사용자와 상호작용-사용자의 요구사항에 따라 대응 가능함. 예시: 클릭하면 화면에 표시된다
- 구조 스타일 변경-자바스크립트를 사용하면 html, css의 구조를 변경 가능. 예시: 버튼 클릭시 색상 변경
- 인터렉티브 애니메이션-사용자와 상호작용으로 애니메이션 적용 가능
- 보다 다양한 처리-브라우저에서 동적으로 작동하는 것만 아니라, 서버환경,데이터베이스 환경에서도 사용 가능

🏷️요약 설명  
동적 타입 : 변수의 타입을 런타임에 결정(실시간으로 결정)-타입스크립트가 동적 타입에서의 단점을 보충해줌

이벤트 기반: ui에서 특정 이벤트에서 어떤 코드를 실행하겠다

## **파일 연결**

링크 코드는 head영역에만 넣어야함

스크립트 코드는 위치는 <html>밖에는 안됨. <!doctype>의 위나 아래도 안됨.head나 body의 아래에 넣어도 안됨.

반드시 헤드나 바디 **안에** 있어야함.

- 헤드 맨 위에 넣으면 파일 로딩 속도가 느려짐, 가능하면 헤드의 맨 아래의 넣어주면 좋음. 가장 빠른거는 바디의 끝자락에 위치하는 것.
- defer속성이 있으면 헤드 끝자락에 있어도 처리 시간이 빠름. defer는 코드가 위에 있다 할지라도 바디를 다 해석한 다음에 defer를 실행함.
- defer은 dom이 다 실행 되고 난 후 스크립트 코드 실행 시킨다
  -> 이유 : 상단에 적으면 스크립트가 실행이 끝날 때까지 사용자는 흰 화면만 보고 있어야 하기 떄문에 head에서는 하단 그리고 defer 속성을 적어야 한다

## **프론트엔드와 백엔드**

자바스크립트는 본래 프론트엔드 개발에서만 작동하는 언어였지만, 오늘날에는 백엔드 개발에도 사용

### **프론트엔드**

사용자가 보고 만지고 느낄 수 있는 모든 것

웹 브자우저에 존재하는 모든 것은 프론트엔드 기술로 분류

프론트엔드는 클라이언트 측(Client-side)

- 웹페이지 디자인과 레이아웃
- 버튼, 폼, 메뉴 등 사용자 인터페이스
- 애니메이션과 시각 효과
- 사용자 상호작용처리
- 반응형 디자인

### **백엔드**

사용자에게 보이지 않게 숨겨진 것들을 포괄

서버(클라이언트에 표시할 내용을 알려주는 것)와 데이터베이스(데이터를 저장하는 장소)가 포함

백엔드= 컴퓨터, 서버 측 (Server-side)

프론트엔드에서 데이터베이스관리, 사용자 인증, 권한 관리에 관한 서비스를 가져와서 사용 가능

- 데이터베이스 관리 (필요한 것을 요청하고 가져오기)
- 사용자 인증 및 권한 관리 (회원가입, 로그인)
- 비즈니스 로직 처리
- 이메일 발송
- 결제 처리
- API 제공 (클라이언트측에서 사용할 기능을 제공)

## **프론트엔드 JavaScript**

Web API(사용 가능한 기능 모음)라고 통칭되는 함수 집합을 통해 웹 브라우저와 상호작용

### **프론트엔드 에코시스템**

주요 코드 리소스 : 라이브러리, 프레임워크, 플러그인

| **구분**                     | **비유 (건축)** | **설명**                                                                                              |
| ---------------------------- | --------------- | ----------------------------------------------------------------------------------------------------- |
| **자바스크립트 (언어)**      | **기초 자재**   | 모든 것의 기본이 되는 프로그래밍 언어입니다.                                                          |
| **라이브러리 (Library)**     | **가구/소품**   | 내가 필요할 때 골라서 쓰는 부품 모음입니다 (예: jQuery, React).                                       |
| ⇒ 내가 주도권을 가짐         |
| **프레임워크 (Framework)**   | **모델하우스**  | 이미 짜인 틀 안에서 내가 내용을 채워 넣는 구조입니다 (예: Vue, Angular). 대형 어플리케이션을 만들 때. |
| ⇒ 프레임워크가 주도권을 가짐 |
| **플러그인 (Plugin)**        | **추가 기능**   | 특정 소프트웨어에 끼워 넣어 기능을 확장하는 부품입니다.                                               |

이 도구들은 결국 **자바스크립트라는 엔진**을 더 효율적으로 돌리기 위해 존재한다.

1. **프레임워크**가 전체적인 **시스템의 뼈대와 규칙**을 잡고 (시스템의 흐름 제어)
2. 그 안에서 복잡한 계산이나 반복적인 UI 작업은 **라이브러리**를 가져와서 해결한다.
3. 기존 도구에 없는 특수한 기능(예: 슬라이더, 차트)은 **플러그인**을 꽂아 기능을 보강한다.
4. \*브라우저(엔진)\*\*는 이 모든 것이 조합된 자바스크립트 코드를 해석하여 사용자에게 화면을 보여준다.

### 웹사이트를 만드는 과정

웹사이트 제작 과정은 크게 **설계, 개발, 배포**의 단계로 나뉩니다. 수업 내용을 바탕으로 자바스크립트와 도구들이 각 단계에서 어떻게 쓰이는지 정리해 드릴게요.

### 1. 설계 및 환경 구축 (Framework)

- 먼저 웹사이트의 뼈대를 잡습니다.
- \*프레임워크(React, Vue 등)\*\*를 선택하여 프로젝트의 전체적인 구조와 규칙을 설정합니다.

### 2. UI/UX 구현 (JavaScript + Library)

- **자바스크립트**로 브라우저에 보여줄 데이터를 처리하고 동적인 기능을 만듭니다.
- 버튼 클릭 시 메뉴가 나오거나, 화면 전환 효과 등은 **라이브러리**나 **플러그인**을 가져와 효율적으로 구현합니다.

### 3. 코드 최적화 및 압축 (Prettier + Minification)

- 작업이 끝나면 **프리티어**로 코드를 예쁘게 정리합니다.
- 파일 용량을 줄여 웹사이트 로딩 속도를 높이기 위해 **코드 압축(Minification)** 과정을 거칩니다.

### 4. 배포 (Netlify 등)

- 완성된 코드를 **네트리파이** 같은 호스팅 플랫폼에 업로드합니다.
- 이때 **Web API**를 통해 서버와 통신하며 실제 데이터가 흐르는 완성된 웹사이트가 사용자에게 제공됩니다.

### 플러그인은 언제 사용하는것인가요 ?

- \*플러그인(Plugin)**은 이미 완성된 소프트웨어나 프레임워크에 **특정한 추가 기능만 '톡' 끼워 넣고 싶을 때\*\* 사용합니다.
  ### 1. 언제 사용하나요?
  - **기능 확장:** 원래 프로그램에는 없던 새로운 기능을 추가하고 싶을 때 사용합니다.
  - **반복 작업 방지:** 누군가 이미 잘 만들어놓은 특정 기능(예: 이미지 슬라이더, 달력)을 내 웹사이트에 바로 가져다 쓰고 싶을 때 사용합니다.
  - **에디터 강화:** VS Code 자체에는 코드 압축 기능이 없지만, 'Minify 플러그인'을 설치해 그 기능을 추가하는 것이 대표적인 예시입니다.

| **사용처**   | **플러그인 예시** | **하는 역할**                             |
| ------------ | ----------------- | ----------------------------------------- |
| **웹사이트** | Swiper            | 마우스로 넘기는 이미지 슬라이드 기능 추가 |
| **VS Code**  | Prettier          | 코드를 예쁘게 정렬해주는 기능 추가        |
| **브라우저** | AdBlock           | 웹 페이지의 광고를 차단하는 기능 추가     |
