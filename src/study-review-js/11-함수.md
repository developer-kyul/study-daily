# 함수 (Functions)

함수(기능) 선언

```jsx
function 함수정의(파라미터1, 파라미터2) {
  return `${파라미터1}가 파라미터로 들어왔어요`;
}

// 함수 호출 (기능 사용)
console.log(함수정의('인자(아규먼트)'));
```

### 함수 이름 정의, 함수의 매개변수 나열, 함수의 결과 반환

함수는 특정 순서에 따라 작업을 실행하는 코드 블록이다.

### 함수(기능)

함수란? 값을 담는 상자이며 "어떤 입력이 주어지면 결과를 내는 구조"

마치, 자판기(동전→음료)나 믹서기(과일→주스)처럼 ‘작동’ 한다.

> 입력(Input) ⇒ 처리(Processing) ⇒ 출력(Output)

1. **입력 :** "이번엔 큰 양동이 가져가!" (매개변수 전달)
2. **처리 :** 정해진 순서대로 우물에서 물을 긷는다. (함수 내부 로직 실행)
3. **출력 :** "여기 깨끗한 물 한 통!" (반환값 전달)

## 함수 구문

```jsx
function 함수이름(매개변수) {
  // 함수 내부에 코드 작성
}
```

`function` 키워드는 JavaScript 엔진에 함수를 선언한다는것

함수 이름은 사용자가 임의로 작성할 수 있고, 의미있는 이름으로 작성

## 함수 이름

함수 이름 작성 규칙은 변수 이름 작성 규칙과 유사

1. 한 단어로 구성되어야 하며, 공백은 허용되지 않음
2. 알파벳(`a-z`, `A-Z`), 숫자(`0-9`), 특수 문자(`$`, `_`) 조합으로 구성
3. 변수와 마찬가지로, 숫자로 시작하는 함수 이름은 허용되지 않습니다.
4. [예약 키워드(reserved words)](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Lexical_grammar#reserved_words)는 함수 이름으로 사용될 수 없음

함수를 선언한 후에 함수 이름 뒤에 소괄호(`()`)를 붙여 함수를 사용(호출 또는 실행)할 수있다.

매개변수와 리턴값은 선택사항이다 !

함수이름 뒤에 실행 연산자인 소괄호 ()를 붙이면 함수가 실행

## 들여쓰기

함수 블록 안의 코드(중괄호(`{}`) 안에 있는 모든 코드)는 오른쪽으로 들여쓰기

들여쓰기는 사람이 코드를 더 읽기 쉽게 도와줍니다. (일관성 유지 필요)

## 매개변수

"매개변수(Parameters)"는 선택 사항

매개변수는 함수에 선언할 변수를 쉼표(`,`)로 구분하여 나열한 것

매개변수는 함수를 사용할 때 값을 지정가능

함수 박스안에서만 사용할 수 있는 변수를 매개변수라고 한다.

함수는 매개변수(Parameters)를 받을 수 있으며, 매개변수란? "함수에 선언하려는 변수"를 의미한다.

매개변수는 함수 내부에서 사용될 수 있습니다. (매개변수 = 함수 내부의 지역 변수)

각 매개변수는 쉼표(`,`)로 구분

## 인자

매개변수는 사용하기 전에 값을 지정해야 합니다.

따라서 함수를 사용할 때 "인자(arguments)"라고 불리는 값을 전달할 수 있다.

```jsx
함수이름('값1', '값2');
```

# 반환문

함수는 `return` 키워드를 사용해 "반환문(return statement)"을 설정할 수

JavaScript 엔진은 함수가 사용될 때 `return` 키워드를 확인하면 실행을 중단하고 값을 반환한다.

표현식을 평가한 값, 또는 값을 반환문으로 결과 도출 / 함수 종료(뒤에 있는 코드는 모두 무시 중단)

# 표현식

함수 표현식

JavaScript에서 표현식(expression)은 전달되기 전에 평가되어야 하는 값으로 정의

숫자를 함수에 전달하면 즉시 사용할 수 있다.

- 문자, 숫자, 불리언, 함수 등 → 바로 전달 가능 (값이므로)
- 표현식이나 복잡한 계산의 중간 상태 → 먼저 계산(평가)한 후 전달 가능
- 함수 선언과 / 함수 표현식의 차이 확인하기

### 익명 함수

- **일회성 작업**: 한 번만 쓰고 버릴 함수에 굳이 이름을 지어줄 필요가 없을 때 유용합니다.
- **메모리 효율**: 이름을 가진 함수는 메모리에 계속 상주하지만, 익명 함수는 필요한 순간에 생성되어 사용된 후 사라지기 좋습니다.
